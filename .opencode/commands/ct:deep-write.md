对指定词条执行深度写作全流程：自动识别词条类型 → 匹配写作策略 → 生成大纲 → 撰写正文 → 质量审查 → 自动修复。

如果用户未指定词条，询问：

1. 为哪个词条执行深度写作？
2. 目标深度层级是什么？（可选，默认从 frontmatter depth 字段读取）
   - `beginner`：生活化类比 + 无公式 + 1000-1500 字
   - `intermediate`：关键公式/模型 + 案例 + 1500-3000 字
   - `advanced`：完整数学模型 + 代码示例 + 历史演变 + 3000-5000 字
3. 是否覆盖已有内容？（如有内容，先确认）

## 执行步骤

### 步骤 0：确定深度层级

读取词条文件的 frontmatter，检查 `depth` 字段。如果不存在或用户指定了目标深度，以用户指定为准。深度层级影响大纲的章节数量和复杂度（详见每个策略下方的"按 depth 扩展"规则）。

如果没有 depth 字段且用户未指定，默认为 `beginner`，完成后更新 frontmatter 添加 `depth` 字段。

### 步骤 1：识别词条类型

读取 `docs/glossary/index.md`，根据词条所属分类确定写作策略：

| 分类                     | 写作策略     |
| ------------------------ | ------------ |
| 基础概念、经济学经典理论 | 知识科普     |
| 金融学核心理论           | 理论深度解析 |
| 货币与银行、投资市场     | 体系讲解     |
| 衍生品与风控             | 原理+实战    |
| 公司金融                 | 实务指南     |
| 宏观经济、国际金融       | 宏观视角     |
| 个人理财                 | 实操指南     |
| 金融科技、新金融概念     | 前沿解读     |
| 金融史与重大危机         | 案例分析     |
| 金融监管与机构           | 制度解读     |
| 投资流派与方法论         | 方法论对比   |
| 量化投资                 | 技术深度解析 |
| 工具资源                 | 选型指南     |

### 步骤 2：生成大纲

根据写作策略生成对应结构的大纲，要求：

- 使用 Markdown 标题层级（## 一级章节，### 二级章节）
- 每个章节包含简短说明和建议字数
- 标注需要图表或公式的位置
- 列出需要交叉引用的相关词条（3-5 个）

### 步骤 3：撰写正文

按大纲逐章节撰写，要求：

- **专业性**：概念描述准确，数据引用最新
- **通俗性**：使用生活化类比解释复杂概念
- **可读性**：段落简短（不超过 5 行），逻辑清晰
- **术语规范**：金融术语首次出现标注英文
- **对比分析**：对比类词条必须包含方案对比表格
- **交叉引用**：内部链接使用相对路径（如 `/investing/stock`）
- **延伸阅读**：外部链接指向权威资料
- **数据来源标注**（强制）：文中出现的每一个具体数据/数字/统计结果，必须在其后标注来源。格式：`（来源：机构名称，年份）`。例如：
  - "美国 CPI 在 2022 年 6 月达到 9.1%（来源：美国劳工统计局，2022）"
  - "中国 M2 余额在 2024 年底突破 300 万亿元（来源：中国人民银行，2024）"
  - 不含具体数据的定性描述（如"通货膨胀是物价持续上涨的现象"）无需标注
  - 标注不完整的数据视为未完成，会在审查阶段标记

### 步骤 4：质量审查

审查维度：

1. **内容准确性**：金融概念描述正确，无过时信息
2. **内容完整性**：覆盖大纲所有章节，无未完成段落
3. **语言一致性**：统一中文，术语前后一致，首次标注英文
4. **格式规范性**：标题层级正确，链接格式统一
5. **关联性**：包含 3-5 个相关词条链接，延伸阅读指向相关专题
6. **数据溯源**：检查文中所有具体数据/数字/统计结果是否标注了来源（`（来源：...）`）格式。缺失标注的数据逐一列出

### 步骤 5：自动修复

自动修复以下问题：

- 链接格式不一致
- 金融术语首次出现未标注英文
- 提示块使用不当
- 数据来源标注格式不一致（统一为 `（来源：机构名称，年份）` 格式）

对于无法自动补全数据来源的情况（如 AI 不确定具体年份或机构），在报告中标记为待人工确认。

## 写作策略详解

### 知识科普（基础概念、经济学经典理论）

基础模板（适用于所有 depth）：

```markdown
## 概述

- 核心定义（100-150 字）
- 一句话通俗解释

## 演进历史

- 起源与发展脉络
- 关键里程碑

## 核心分类/类型

- 分类 1：说明 + 举例
- 分类 2：说明 + 举例

## 为什么重要

- 在金融体系中的地位
- 对普通人的影响

## 与其他概念的关系

- 链接到相关词条（3-5 个）

## 延伸阅读

- 权威书籍、论文、参考资料
```

按 depth 扩展：

- `beginner`：使用基础模板，无额外章节，总字数 1000-1500
- `intermediate`：增加"常见误区"章节（3 个误区 + 纠偏），增加 1 个数据图表（如趋势图），总字数 1500-2500
- `advanced`：增加"学术争论"章节（不同学派观点对比），增加"数据与方法"章节（定量分析），总字数 2500-4000

### 理论深度解析（金融学核心理论）

基础模板（适用于所有 depth）：

```markdown
## 概述

- 核心定义与理论定位
- 解决了什么问题

## 理论前提

- 基本假设
- 适用条件

## 核心公式/模型

- 公式说明（配图）
- 各变量含义
- 计算示例

## 理论意义

- 理论创新点
- 对金融实践的影响

## 局限与批评

- 主要争议
- 现实中的偏离

## 与其他概念的关系

- 链接到相关词条（3-5 个）

## 延伸阅读

- 原始论文、经典教材、权威解读
```

按 depth 扩展：

- `beginner`：仅保留概述 + 理论意义 + 局限，去掉公式/模型的详细推导，用通俗语言解释核心思想，总字数 1000-1500
- `intermediate`：使用基础模板，每个公式必须配计算示例，增加 1 个实证研究引用，总字数 1500-3000
- `advanced`：增加"数学推导"章节（完整公式推导过程），增加"实证检验"章节（相关研究结果），增加"理论变体"章节（如 CAPM 的扩展模型），总字数 3000-5000

### 体系讲解（货币与银行、投资市场）

基础模板（适用于所有 depth）：

```markdown
## 概述

- 是什么 + 解决什么问题
- 在金融市场中的角色

## 核心参与者

- 参与者 1：角色 + 作用
- 参与者 2：角色 + 作用

## 运作机制

- 流程说明
- 关键环节

## 主要产品/工具

- 产品 1：特点 + 示例
- 产品 2：特点 + 示例

## 风险与注意事项

- 主要风险
- 普通人需要注意什么

## 与其他概念的关系

- 链接到相关词条（3-5 个）

## 延伸阅读

- 权威资料、学习资源
```

按 depth 扩展：

- `beginner`：使用基础模板，每个产品/工具配 1 个生活化类比，总字数 1000-1500
- `intermediate`：增加"行业对比"章节（如中美市场对比），核心运作机制配流程图说明，总字数 1500-3000
- `advanced`：增加"监管框架"章节，增加"量化指标"章节（关键数据指标及其意义），增加"历史演变"子章节，总字数 3000-5000

```markdown
## 与其他概念的关系
```

### 案例分析（金融史与重大危机）

```markdown
## 概述

- 事件背景
- 基本信息

## 时间线

- 关键时间节点
- 事态演变

## 原因分析

- 直接原因

## 影响与后果

- 短期影响

## 历史教训

- 总结了哪些经验

## 与其他概念的关系
```

## 工具绑定

| 步骤                | 工具                           |
| ------------------- | ------------------------------ |
| 读取词条和 index.md | Read                           |
| 写入大纲/正文       | Write（新文件）或 Edit（追加） |
| 统计字数            | Grep 排除 frontmatter          |
| 自动修复            | Edit                           |
| 验证                | Bash 运行 pnpm 命令            |

## 容错

- 词条文件不存在 → 先用 `/ct:propose` 创建
- 分类不在映射表中 → 使用"知识科普"作为默认策略
- `docs:build` 失败 → 读取错误日志，修复后重试（最多 3 次）
- 正文字数超预期过多（> 阈值 150%）→ 建议拆分为专题

示例：`/ct:deep-write 期权` 或 `/ct:deep-write 量化投资`
