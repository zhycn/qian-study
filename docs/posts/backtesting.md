# 回测

回测就是把你的交易策略放到历史数据里"模拟考试"——考得好不代表实盘能赚钱，但考不好实盘一定亏。

## 概述

回测（Backtesting）就是把你的交易策略放到过去的历史数据中跑一遍，看看如果按照这个策略操作，能赚多少钱、亏多少钱、最大回撤有多大。就像开车前先看导航预估时间和路线一样，回测是量化投资者在真金白银投入前必须做的"模拟考"。

回测的核心价值不是"证明策略能赚钱"，而是**"发现策略会亏钱的地方"**。一个好的回测应该尽可能暴露策略的弱点，而不是美化策略的表现。

## 为什么重要

在量化投资中，一个策略听起来再完美，没有经过历史数据验证都是空中楼阁。回测能帮你：

- 验证策略逻辑是否成立
- 评估策略在不同市场环境下的表现
- 发现策略的潜在风险和缺陷
- 优化参数，找到最优配置
- 估算策略容量（Strategy Capacity）

没有回测就实盘交易，等于闭着眼睛开车。但回测结果好也不等于实盘能赚钱——回测和实盘之间隔着交易成本、滑点、流动性等多重障碍。

## 核心原理

回测的数学本质是**对策略在历史数据上的收益分布进行估计**：

$$\hat{E}[R_{\text{strategy}}] = \frac{1}{T} \sum_{t=1}^{T} R_t(\text{Strategy} | \text{Data}_{1:t-1})$$

关键约束：在时刻 $t$，策略只能使用时刻 $t$ 之前的信息（$\text{Data}_{1:t-1}$）。

### 回测的统计显著性

一个策略在回测中表现好，可能是运气而非实力。如何判断？

**t 检验**：

$$t = \frac{\bar{R}}{\sigma_R / \sqrt{T}}$$

其中 $\bar{R}$ 是平均收益，$\sigma_R$ 是收益标准差，$T$ 是样本数。

**但这里有个大问题**：如果你测试了 100 个策略，其中表现最好的那个，其 t 统计量会天然偏高。这就是**多重检验问题（Multiple Testing Problem）**。

Harvey, Liu & Zhu（2016）（来源：Harvey, Liu & Zhu, "... and the Cross-Section of Expected Returns", Review of Financial Studies, 2016）提出，考虑到多重检验，新的因子需要 t > 3.0 才能被认为是显著的（传统标准是 t > 2.0）。

### 回测中的关键偏差

**1. 幸存者偏差（Survivorship Bias）**

如果回测只使用当前存在的股票数据，会忽略已退市股票的表现。这会导致回测收益被高估，因为退市股票通常是表现差的。

**解决方案**：使用包含已退市股票的全样本数据（Point-in-Time Database）。

**2. 前视偏差（Look-ahead Bias）**

回测时不小心使用了当时不可能知道的信息。例如：

- 用年报数据做决策，但年报的实际发布日期比报告期晚了几个月
- 用当天的收盘价做买入决策，但实际交易时收盘价还不知道

**解决方案**：使用 Point-in-Time 数据，确保每个时点只使用当时可获取的信息。

**3. 过拟合（Overfitting）**

策略在历史数据上表现完美，只是因为"背下了答案"。这是回测中最常见也最危险的偏差。

**解决方案**：

- 样本外测试（Out-of-sample Testing）
- 交叉验证（Cross-Validation）
- 减少自由度（简化策略）
- 经济逻辑验证

## 技术/方法详解

### 回测引擎架构

一个完整的回测引擎包含以下模块：

**1. 数据模块**

- 行情数据：价格、成交量、买卖盘口
- 基本面数据：财务报表、估值指标
- 公司行为数据：分红、拆股、并购
- 数据必须是 Point-in-Time 的

**2. 信号模块**

- 根据策略逻辑生成交易信号
- 信号可以是连续的（强度值）或离散的（买入/卖出）

**3. 组合管理模块**

- 根据信号和仓位管理规则计算目标权重
- 处理约束条件（最小交易单位、行业限制等）

**4. 执行模块**

- 模拟订单执行
- 计算交易成本（手续费、滑点、市场冲击）
- 处理成交（全部成交、部分成交、未成交）

**5. 绩效分析模块**

- 计算收益、波动率、夏普比率、最大回撤等指标
- 生成归因报告

### 核心绩效指标

| 指标           | 公式                                        | 说明                 |
| -------------- | ------------------------------------------- | -------------------- | ----- | -------------- |
| **年化收益率** | $(1 + R_{\text{total}})^{365/T} - 1$ | 策略的年化收益 |
| **年化波动率** | $\sigma_{\text{daily}} \cdot \sqrt{252}$ | 策略的风险水平 |
| **夏普比率** | $\frac{R_p - R_f}{\sigma_p}$ | 风险调整后的收益 |
| **最大回撤** | $\max_t \frac{M_t - P_t}{M_t}$ | 最大峰值到谷值的跌幅 |
| **卡尔玛比率** | $\frac{\text{年化收益}}{\text{最大回撤}}$ | 收益与最大回撤的比 |
| **胜率** | $\frac{\text{盈利交易数}}{\text{总交易数}}$ | 盈利交易的占比 |
| **盈亏比** | $\frac{\text{平均盈利}}{\text{平均亏损}}$ | 盈利与亏损的比 |
| **换手率** | $\frac{\sum \|w_t - w_{t-1}\|}{2}$ | 组合的周转频率 |

### 回测验证方法

**1. 样本分割**

将数据分为训练集（In-sample）和测试集（Out-of-sample）：

```
[--- 训练集 70% ---][--- 测试集 30% ---]
```

在训练集上优化参数，在测试集上验证表现。

**2. 滚动窗口（Walk-Forward Analysis）**

```
[窗口1: 训练][测试][窗口2: 训练][测试]...
```

每次用固定长度的窗口训练，然后在接下来的窗口测试，滚动前进。这更接近实盘的真实情况。

**3. 交叉验证**

K 折交叉验证，将数据分为 K 份，轮流用 K-1 份训练、1 份测试。

**4. 蒙特卡洛模拟**

对策略收益序列进行随机重排，生成 10000 条模拟路径，评估策略表现的统计显著性。

## 实战应用

### 回测检查清单

在相信回测结果之前，逐项检查：

- [ ] 数据是否包含已退市股票（无幸存者偏差）
- [ ] 数据是否为 Point-in-Time（无前视偏差）
- [ ] 是否考虑了分红、拆股等公司行为
- [ ] 是否纳入了交易成本（手续费 + 滑点）
- [ ] 是否考虑了流动性约束
- [ ] 回测周期是否覆盖牛熊（至少 5-10 年（来源：行业最佳实践））
- [ ] 是否做了样本外测试
- [ ] 参数是否经过敏感性分析
- [ ] 策略是否有经济学/行为学逻辑支撑
- [ ] 策略容量是否合理估计

### 从回测到实盘的"衰减系数"

经验上，实盘表现通常是回测表现的 50%-70%（来源：行业经验总结，多个量化机构实证数据）。衰减主要来自：

- **交易成本**：回测中可能低估了滑点和冲击成本
- **信号延迟**：实盘中信号触发到订单执行有延迟
- **流动性**：回测假设订单能按收盘价成交，实盘可能无法做到
- **心理因素**：实盘中面对亏损时的心理压力可能导致不执行策略

### 回测陷阱案例

**案例 1：未来函数**

策略逻辑："如果当天收盘价高于 20 日均线，则当天买入"。问题：你不可能在当天开盘前就知道收盘价。

**案例 2：幸存者偏差**

策略在 2010-2024 年的 A 股数据上回测，年化收益 25%。但数据只包含当前存在的股票，忽略了期间退市的股票。实际收益可能只有 15%（来源：假设性案例，说明幸存者偏差）。

**案例 3：过拟合**

策略有 10 个参数，在 5 年数据上优化后年化收益 40%。但在接下来的 2 年数据上年化收益 -5%。参数太多、数据太少是过拟合的典型原因（来源：假设性案例，说明过拟合风险）。

## 关键要点

- 回测结果不等于未来收益，历史不会简单重复
- 要警惕过拟合，策略在历史数据上表现太好可能是"死记硬背"
- 必须考虑交易成本，包括手续费、滑点和市场冲击
- 样本外测试是检验策略泛化能力的关键
- 回测周期要足够长，覆盖牛熊不同市场阶段
- 回测的核心价值是发现策略的弱点，而不是证明策略能赚钱

## 常见误区

- **"回测收益 = 未来收益"**：历史不会简单重复
- **"忽略交易成本"**：手续费、滑点、市场冲击会让理论收益大幅缩水
- **"回测周期太短"**：回测周期要足够长，覆盖牛熊不同市场阶段
- **"过度优化参数"**：参数优化过度会导致过拟合
- **"未来函数陷阱"**：回测时不小心用了当时不可能知道的信息
- **"回测好就实盘"**：必须经过模拟交易验证

## 中国投资者回测提示

- **A 股特殊规则**：回测需考虑涨跌停限制（10%/20%）、T+1 交易制度、停牌等因素
- **数据源选择**：A 股数据可用 Tushare、AKShare、JoinQuant 等免费数据源
- **复权处理**：A 股分红送配频繁，必须使用前复权或后复权数据
- **交易成本**：A 股印花税（卖出 0.05%）、佣金（万 1-万 3）需纳入回测
- **滑点估计**：A 股小盘股滑点可能较大，建议设置 0.1%-0.3% 的滑点
