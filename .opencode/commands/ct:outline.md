为指定词条生成深度写作大纲。

如果用户未指定词条，询问：

1. 为哪个词条生成大纲？
2. 期望的文章深度？（简短 500 字、标准 1500 字、详细 3000 字）
3. 目标读者是谁？（金融小白、投资者、从业者）

## 执行步骤

### 步骤 1：读取当前内容

用 Read 工具读取词条文件。判断状态：

- 文件不存在 → 询问是否创建（跳转 `/ct:propose`）
- 仅 frontmatter → 替换整个文件
- 已有正文 → 在末尾追加（正文后加 `---` 分隔）
- 已有完整大纲 → 提示用户跳过或重新生成

### 步骤 2：生成大纲结构

根据词条定位选择模板：

| 类型            | 模板                                             |
| --------------- | ------------------------------------------------ |
| 基础概念/理论类 | 定义、历史、分类、应用场景、相关概念             |
| 市场/产品类     | 参与者、运作机制、主要产品、风险、实操建议       |
| 工具/方法类     | 使用场景、实施步骤、最佳实践、常见问题、工具推荐 |
| 历史/事件类     | 背景、时间线、原因、影响、教训                   |

如果无法判断类型，默认使用"基础概念/理论类"模板。

### 步骤 3：大纲要求

- 使用 Markdown 标题层级（`##` 一级章节，`###` 二级章节）
- 每个章节包含简短说明和建议字数
- 标注需要图表或公式的位置（如 `[图表：趋势图]`）
- 用 Grep 搜索相关词条，列出交叉引用（如 Grep 无结果则跳过）

### 步骤 4：写入文件

用 Edit 或 Write 工具将大纲写入词条文件。

### 步骤 5：验证

确认文件写入成功。

## 大纲模板

```markdown
## 概述

- 核心定义（100-200 字）

## 为什么重要

- 在金融体系中的地位和意义（150-250 字）

## 核心原理/分类/组件

- 关键要点 1
- 关键要点 2

## 与其他概念的关系

- [相关词条 1](/path)
- [相关词条 2](/path)

## 延伸阅读

- [专题目录](/path)
```

注意：此模板为初稿大纲。如需更精细的写作策略（含 depth 扩展），请参考 `/ct:deep-write` 中的写作策略详解。

## 输出格式

```markdown
## 大纲完成：[词条名]

- 章节：X 个
- 估计总字数：X 字
- 交叉引用：X 个词条
- 文件写入：✅
```

## 工具绑定

| 步骤         | 工具         |
| ------------ | ------------ |
| 读取词条     | Read         |
| 搜索相关词条 | Grep         |
| 写入大纲     | Edit / Write |

## 容错

- 文件写入失败 → 检查目录权限，或提示用户确认路径
- 词条已有完整大纲 → 提示用户跳过或重新生成
- 文件路径不存在 → 提示用户确认路径或跳转 `/ct:propose`
- Grep 搜索无结果 → 跳过交叉引用

## 示例

- `/ct:outline 期权`
- `/ct:outline 量化投资 --depth 详细`
