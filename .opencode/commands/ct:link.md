维护词条之间的关联和交叉引用。

如果用户未指定任务，询问：
1. 要执行什么操作？（检查断链、添加关联、批量更新、关联推荐）

## 执行步骤

### 操作 1：检查断链

1. 用 Glob 找到 `docs/` 下所有 `*.md` 文件
2. 用 Grep 查找所有 `](/` 模式的内部链接
3. 用 Read 工具逐一验证每个链接目标文件存在
4. 列出不存在的链接

### 操作 2：添加关联

1. 用 Read 工具读取词条内容
2. 识别文中提到的金融概念（但不含链接的）
3. 用 Glob 搜索这些概念对应的文件
4. 用 Edit 工具为匹配的概念添加 `[概念名](/path)`

### 操作 3：批量更新（重命名/移动时）

1. 用 Grep 搜索所有文件中指向旧路径的链接
2. 用 Edit 工具逐个替换为新的路径
3. 同步更新 index.md 和 config.mts 中的引用

### 操作 4：关联推荐

1. 分析词条关键词
2. 用 Grep 在 `docs/` 下搜索相关文件
3. 推荐 3-5 个应关联的词条
4. 推荐延伸阅读指向的专题

## 输出格式

```
## 交叉引用报告

### 断链（X 个）
| 来源文件 | 断链 | 建议修复 |
|----------|------|----------|
| what-is-finance.md | /basics/xxx | 文件不存在 |

### 缺失关联（X 个）
| 来源文件 | 提到概念 | 目标文件 |
|----------|----------|----------|
| 期权.md | 期货 | /derivatives/futures.md |

### 推荐新增延伸阅读
- 量化投资.md → /quant/
```

## 容错

- 无断链 → 输出"未发现断链"
- 无缺失关联 → 输出"关联完整"
- 建议修改前先用输出报告让用户确认再执行
- 批量替换时使用 Edit 工具而非 sed，避免语法错误

## 示例

- `/ct:link --check-broken`
- `/ct:link --add 期权`
- `/ct:link --update <old-path> <new-path>`
