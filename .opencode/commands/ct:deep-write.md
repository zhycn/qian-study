对指定词条执行深度写作全流程：自动识别词条类型 → 匹配写作策略 → 生成大纲 → 撰写正文 → 质量审查 → 自动修复。

如果用户未指定词条，询问：

1. 为哪个词条执行深度写作？
2. 是否覆盖已有内容？（如有内容，先确认）

## 执行步骤

### 步骤 1：识别词条类型

读取 `docs/glossary/index.md`，根据词条所属分类确定写作策略：

| 分类           | 写作策略     |
| -------------- | ------------ |
| 基础概念       | 知识科普     |
| 模型与架构     | 技术深度解析 |
| 核心技术       | 技术深度解析 |
| 交互与工程     | 实战教程     |
| 关键概念       | 知识科普     |
| 工具与基础设施 | 选型指南     |
| Agent 生态     | 实战教程     |
| 工程与实践     | 实战教程     |
| 安全与伦理     | 深度思考     |
| 行业与应用     | 实战教程     |
| 模型优化       | 技术深度解析 |

### 步骤 2：生成大纲

根据写作策略生成对应结构的大纲，要求：

- 使用 Markdown 标题层级（## 一级章节，### 二级章节）
- 每个章节包含简短说明和建议字数
- 标注需要代码示例或图表的位置
- 列出需要交叉引用的相关词条（3-5 个）

### 步骤 3：撰写正文

按大纲逐章节撰写，要求：

- **专业性**：技术描述准确，引用权威来源
- **深度**：不止于表面定义，深入原理和实践
- **可读性**：段落简短（不超过 5 行），逻辑清晰
- **术语规范**：技术术语首次出现标注英文
- **代码示例**：工程类词条必须包含可运行的代码示例
- **对比分析**：架构类词条必须包含方案对比表格
- **交叉引用**：内部链接使用相对路径（如 `/glossary/ai`）
- **延伸阅读**：外部链接指向官方文档、论文、权威资料

### 步骤 4：质量审查

审查维度：

1. **内容准确性**：技术描述正确，无过时信息
2. **内容完整性**：覆盖大纲所有章节，无未完成段落
3. **语言一致性**：统一中文，术语前后一致，首次标注英文
4. **格式规范性**：标题层级正确，代码块标注语言，链接格式统一
5. **关联性**：包含 3-5 个相关词条链接，延伸阅读指向相关专题

### 步骤 5：自动修复

自动修复以下问题：

- 代码块缺少语言标注
- 内部链接格式不一致
- 技术术语首次出现未标注英文
- 提示块使用不当

## 写作策略详解

### 知识科普（基础概念、关键概念）

```markdown
## 概述

- 核心定义（100-150 字）
- 一句话通俗解释

## 演进历史

- 起源与发展脉络
- 关键里程碑

## 核心分类/类型

- 分类 1：说明 + 示例
- 分类 2：说明 + 示例

## 为什么重要

- 在 AI 领域的地位
- 对开发者的意义

## 与其他概念的关系

- 链接到相关词条（3-5 个）

## 延伸阅读

- 官方文档、论文、权威资料
```

```markdown
- 方案对比表格
- 选型建议

## 工程实践

- 代码示例

- 常见问题

## 与其他概念的关系

```
