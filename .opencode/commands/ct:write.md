---
description: 词条内容创作
agent: build
---

根据词条大纲完成正文内容创作。

如果用户未指定词条，询问：

1. 为哪个词条撰写内容？
2. 是否已有大纲？（如无，先调用 `/ct:outline`）
3. 写作风格偏好？（通俗易懂、学术严谨、实战指南）
4. 目标深度层级？（可选 `beginner`/`intermediate`/`advanced`，默认 `beginner`）

## 执行步骤

### 步骤 1：读取已有内容

用 Read 工具读取词条文件。如果文件仅有 frontmatter（正文字数 < 300 字，使用 `awk '/^---$/{c++; next} c==2' 文件 | wc -m` 统计），按新内容写；如果已有正文且未要求覆盖，询问是否继续。

### 步骤 2：按大纲逐章节撰写

使用 Edit 工具（追加模式）写入正文。每完成一个章节后输出简短状态。

撰写要求：

- 使用中文
- 结构清晰，使用多级标题
- 金融术语首次出现时标注英文 `（英文）`
- 适当使用提示块（tip、warning、info）
- 关键概念加粗
- 内部链接使用相对路径（如 `/basics/what-is-finance`）
- 外部链接使用完整 URL
- 使用生活化类比和举例解释复杂概念
- **数据来源标注（强制）**：文中每一个具体数据/数字/统计结果必须标注来源格式 `（来源：机构名称，年份）`

根据词条分类选择对应的写作策略（参考 `/ct:deep-write` 中的"写作策略详解"部分）。

### 步骤 3：内容质量标准

逐项自检：

- 准确性：金融概念描述正确，数据无过时
- 完整性：覆盖大纲所有章节
- 可读性：段落简短（<= 5 行），多用类比和举例
- 关联性：包含 3-5 个相关词条链接
- 数据溯源：所有具体数据必须有 `（来源：...）` 标注

### 步骤 4：补充结尾

完成后在末尾添加：

- 相关词条链接列表
- 延伸阅读链接（指向专题目录）

### 步骤 5：验证

按 AGENTS.md 验证顺序运行：`pnpm lint:check` → `pnpm type-check` → `pnpm docs:build`。

## 输出格式

```markdown
## 写作完成：[词条名]

### 统计

- 章节数：X 个
- 正文字数：约 X 字
- 内部链接数：X 个
- 数据来源标注：X 处

### 验证

- lint:check ✅/❌
- type-check ✅/❌
- docs:build ✅/❌
```

## 工具绑定

| 步骤     | 工具             |
| -------- | ---------------- |
| 读取词条 | Read             |
| 撰写正文 | Edit（追加模式） |
| 质量自检 | Read + Grep      |
| 验证     | Bash             |

## 容错

- 文件路径不存在 → 提示用户确认路径
- `pnpm docs:build` 失败 → 读取错误输出，修复后重试（最多 3 次）
- 内容超过 5000 字 → 拆分为专题（提示用户使用 `/ct:expand`）
- 大纲与正文关系：如果是追加模式，在正文后添加 `---` 分隔符再写新内容

## 示例

- `/ct:write 量化投资`
- `/ct:write 期权 --style 通俗`
