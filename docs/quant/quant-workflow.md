# 量化投资流程

> 量化投资不是一拍脑袋就写代码，而是一个从"灵光一闪"到"真金白银"的系统化工程。

## 概述

一个量化策略从诞生到上线，需要经历一套完整的流程。这个过程就像把一道"创意菜"变成"标准化菜品"：你需要反复试验、调整配方、内部试吃、小范围公测，最后才能正式上架。

典型的量化投资流程包括六个阶段：**想法生成 → 数据准备 → 策略开发 → 回测验证 → 模拟交易 → 实盘运行**。每个环节都有其关键任务和常见陷阱。

在成熟机构中，一个策略从想法到实盘平均需要 3-12 个月（来源：行业调研数据），只有不到 5% 的初始想法能最终上线。

## 为什么重要

理解量化投资的全流程，能帮你：

- **建立系统化思维**：量化不是"写个代码就跑"，而是需要严谨的工程化流程
- **避开常见坑**：很多新手在回测阶段就犯了"过度拟合"的错误，实盘后才发现策略根本不行
- **评估量化产品**：知道一个靠谱的策略应该经历哪些验证环节，才能判断它是否值得信赖
- **合理分配时间**：在量化投资中，数据清洗和回测验证的时间往往远多于策略开发本身

## 核心原理

量化投资流程的核心是**科学方法论**的金融应用：

1. **提出假设**：市场中存在某种可被利用的规律
2. **收集证据**：用数据验证假设是否成立
3. **构建模型**：将规律转化为可执行的交易规则
4. **实验验证**：在历史数据和模拟环境中测试
5. **得出结论**：策略是否值得投入实盘
6. **持续监控**：实盘后持续验证假设是否依然成立

这个流程的关键在于**可证伪性**（Falsifiability）：一个好的量化策略必须能够被数据证伪。如果一个策略"无论什么结果都能解释"，那它就不是科学，而是伪科学。

## 技术/方法详解

### 阶段一：想法生成（Idea Generation）

一切量化策略的起点是一个"可验证的投资逻辑"。想法可以来自：

- **学术文献**：动量效应（Jegadeesh & Titman, 1993（来源：Journal of Finance））、价值溢价（Fama & French, 1992（来源：Journal of Finance））、低波异常（Baker et al., 2011（来源：Journal of Financial Economics））
- **市场观察**：某个现象反复出现，比如"月末效应"、"周一效应"、"财报公告后漂移"（PEAD）
- **另类数据**：卫星图像看停车场车流量、社交媒体情绪分析、信用卡消费数据
- **经验直觉**：资深交易员的盘感，但需要转化为可量化的规则

**关键原则**：想法必须有经济学或行为学逻辑支撑，不能纯粹"数据挖掘"。纯粹的数据挖掘会产生大量"伪规律"——在历史数据中显著，但样本外完全失效。

### 阶段二：数据准备（Data Preparation）

"垃圾进，垃圾出"（Garbage In, Garbage Out）是量化投资的第一定律。

**数据获取**：

- 行情数据：价格、成交量、买卖盘口（Order Book）
- 基本面数据：财务报表、估值指标、宏观经济数据
- 另类数据：新闻、社交媒体、卫星图像、供应链数据

**数据清洗**是量化投资中最耗时但最重要的环节：

- **缺失值处理**：前值填充、插值、删除
- **异常值检测**：3σ 原则、MAD（中位数绝对偏差）、Winsorize（缩尾处理）
- **幸存者偏差（Survivorship Bias）**：使用包含已退市股票的全样本数据
- **前视偏差（Look-ahead Bias）**：确保回测时只使用当时可获取的信息
- **复权调整**：正确处理分红、拆股等公司行为

### 阶段三：策略开发（Strategy Development）

将投资逻辑转化为可执行的算法：

**信号生成**：
$$\text{Signal}_t = f(\text{Data}_t, \text{Parameters})$$

信号可以是连续的（-1 到 +1 的强度值）或离散的（买入/持有/卖出）。

**仓位管理**：

- 等权分配：每个标的分配相同权重
- 波动率倒数加权：波动率低的标的分配更多权重
- 风险平价（Risk Parity）：使每个标的对组合风险的贡献相等
- Kelly 公式：最优仓位比例 $f^* = \frac{p \cdot b - q}{b}$，其中 $p$ 是胜率，$q = 1-p$，$b$ 是盈亏比

**组合优化**：
$$\max_w \left( w^T \mu - \frac{\lambda}{2} w^T \Sigma w \right)$$

其中 $w$ 是权重向量，$\mu$ 是预期收益，$\Sigma$ 是协方差矩阵，$\lambda$ 是风险厌恶系数。

### 阶段四：回测验证（Backtesting）

用历史数据检验策略表现。回测不是"跑一下看看收益多少"那么简单，需要系统性地检验：

**核心指标**：

- 年化收益率（Annualized Return）
- 年化波动率（Annualized Volatility）
- 夏普比率（Sharpe Ratio）：$SR = \frac{R_p - R_f}{\sigma_p}$
- 最大回撤（Maximum Drawdown）
- 卡尔玛比率（Calmar Ratio）：年化收益 / 最大回撤
- 胜率（Win Rate）和盈亏比（Profit/Loss Ratio）

**必须检验的偏差**：

- 过拟合（Overfitting）：策略在训练集上表现好，在测试集上表现差
- 未来函数（Look-ahead Bias）：回测时使用了当时不可知的信息
- 幸存者偏差（Survivorship Bias）：只使用当前存在的股票数据
- 交易成本：手续费、滑点、市场冲击

**交叉验证方法**：

- 样本外测试（Out-of-sample Testing）：将数据分为训练集和测试集
- 滚动窗口（Rolling Window）：用前 N 年数据训练，后 M 年测试，滚动进行
- 交叉验证（Cross-Validation）：K 折交叉验证，提高统计显著性

### 阶段五：模拟交易（Paper Trading）

在真实市场环境中"纸上交易"，检验：

- 信号在实际行情中是否能及时触发
- 订单能否按预期价格成交
- 系统稳定性和延迟表现
- 极端行情下的系统行为

模拟交易通常运行 3-6 个月，期间策略不做任何参数调整。

### 阶段六：实盘运行与风控（Live Trading & Risk Management）

策略上线后，真正的考验才开始：

**实时监控**：

- 策略表现是否偏离回测预期？
- 信号触发频率是否正常？
- 成交率和滑点是否在预期范围内？

**风险控制**：

- 日亏损限额：单日亏损超过阈值时暂停交易
- 最大回撤限额：策略回撤超过历史最大回撤的 1.5 倍时减仓
- 仓位上限：单一标的或行业的最大暴露限制
- 流动性监控：标的流动性下降时自动减仓

**策略迭代**：

- 定期（如每季度）评估策略表现
- 当策略连续 N 个月跑输基准时，考虑调整或下线
- 市场环境变化时，重新训练模型或调整参数

## 主流方案对比

| 阶段     | 个人投资者             | 小型量化团队       | 大型量化机构           |
| -------- | ---------------------- | ------------------ | ---------------------- |
| 想法生成 | 学术论文、论坛         | 内部研究、学术合作 | 专职研究团队、另类数据 |
| 数据     | 免费/低价数据源        | 商业数据供应商     | 自建数据平台、独家数据 |
| 回测框架 | 开源框架（Backtrader） | 自研框架           | 高性能分布式回测系统   |
| 模拟交易 | 量化平台模拟盘         | 券商模拟接口       | 专用模拟交易系统       |
| 实盘     | 券商 API               | 专业交易系统       | 低延迟交易系统、托管   |
| 周期     | 1-3 个月               | 3-6 个月           | 6-12 个月              |

## 实战应用

### 个人投资者的最小可行流程

对于刚入门的量化投资者，可以简化为以下流程：

1. **想法**：从一个简单的逻辑开始（如"低 PE 股票长期跑赢高 PE 股票"）
2. **数据**：使用 Tushare、AKShare 等免费数据源
3. **回测**：用 JoinQuant 或 RiceQuant 在线平台
4. **验证**：至少做 5 年样本外测试
5. **模拟**：用模拟账户跑 1-3 个月
6. **小资金实盘**：用不超过总资金 10% 的金额试水

### 策略上线检查清单

- [ ] 回测周期覆盖至少一个完整的牛熊周期
- [ ] 样本外测试表现与样本内一致（衰减不超过 30%）
- [ ] 交易成本已纳入回测（手续费 + 滑点）
- [ ] 最大回撤在可承受范围内
- [ ] 策略逻辑有经济学/行为学解释
- [ ] 已进行敏感性分析（参数小幅变化不影响结论）
- [ ] 模拟交易运行至少 1 个月
- [ ] 风控规则已设定并测试

## 关键要点

- 量化投资是一个"假设 → 验证 → 执行 → 监控"的闭环过程
- 回测收益 ≠ 实盘收益，中间隔着交易成本、滑点和市场冲击
- 过拟合是量化策略的头号杀手，样本外测试和交叉验证必不可少
- 风控不是"附加项"，而是贯穿全流程的核心环节
- 数据清洗的时间往往远多于策略开发的时间，但这是最值得投入的

## 常见误区

- **"回测收益高 = 实盘能赚钱"**：回测收益往往忽略了交易成本、滑点和市场冲击，实盘表现通常大打折扣
- **"跳过模拟交易直接实盘"**：模拟交易是检验策略在真实市场环境中表现的关键环节
- **"过度依赖历史数据"**：市场结构会变化，过去有效的策略未来不一定有效
- **"忽视数据质量"**：使用有幸存者偏差、未复权或含异常值的数据，回测结果毫无参考价值
- **"策略上线后不监控"**：市场在变，策略需要持续监控和迭代，不是一劳永逸的
- **"频繁调整参数"**：策略上线后频繁调参是过拟合的另一种形式
