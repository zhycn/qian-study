针对多个词条执行批量深度写作全流程：批量识别类型 → 匹配策略 → 大纲 → 写作 → 审查 → 修复。

如果用户未指定范围，询问：

1. 批量处理哪些词条？
   - `empty`：所有仅有 frontmatter 但正文为空的词条
   - `category:xxx`：指定分类的所有词条（如 `category:基础概念`）
   - 用户手动指定列表（以逗号分隔）
2. 目标深度层级是什么？（默认 `beginner`，可选 `intermediate` / `advanced`）
3. 是否覆盖已有内容？（默认不覆盖，仅写入空壳词条）

## 工作流程

### 步骤 1：确定目标词条列表

根据用户指定的范围确定词条列表：

- `empty`：扫描 `docs/` 下所有 `.md` 文件，识别仅有 frontmatter 但正文不足 300 字的词条
- `category:xxx`：从 `docs/glossary/index.md` 读取对应分类下的所有词条
- 手动列表：直接使用用户提供的词条名列表

输出目标词条清单供用户确认。

### 步骤 2：批量识别词条类型

对每个目标词条：

1. 读取 `docs/glossary/index.md`，根据分类确定写作策略
2. 读取词条文件 frontmatter 的 `depth` 字段
3. 如果词条已有内容（> 300 字）且用户未要求覆盖，跳过

### 步骤 3：逐词条执行深度写作

对每个目标词条，按以下顺序执行：

1. 生成大纲（根据写作策略 + depth 层级）
2. 撰写正文
3. 质量审查（8 维度评分，参考 `/ct:review`）
4. 自动修复（链接格式、术语标注、提示块）

每完成一个词条输出简短状态。

### 步骤 4：生成批量报告

```
## 批量深度写作完成

### 处理概况
- 目标词条: X 个
- 成功写入: X 个
- 跳过（已有内容）: X 个
- 失败: X 个

### 逐词条结果
| 词条 | 写作策略 | depth | 字数 | 审查评分 | 状态 |
|------|----------|-------|------|----------|------|
| [词条名](/path) | 知识科普 | beginner | 1,200 | 4.2/5 | ✅ |
| [词条名](/path) | 理论深度解析 | intermediate | 2,400 | 4.5/5 | ✅ |
| ... | ... | ... | ... | ... | ... |

### 验证
- lint:check ✅/❌
- type-check ✅/❌
- docs:build ✅/❌
```

### 步骤 5：提交确认

询问用户是否需要：

- 通过 `/vp:commit-push` 提交变更
- 对特定词条执行 `/ct:review` 深度审查
- 对特定词条执行 `/ct:link` 交叉引用补充

## 注意事项

- 批量操作可能耗时较长，每次处理建议不超过 10 个词条
- 用户可随时中断，已完成的词条不会丢失
- 跳过已有内容的词条，除非用户明确要求覆盖
- 运行完成后依次执行 `pnpm lint:check`、`pnpm type-check`、`pnpm docs:build` 验证

## 示例

- `/ct:batch-write empty` — 补齐所有空壳词条
- `/ct:batch-write category:新金融概念` — 批量写入新金融概念分类下的所有词条
- `/ct:batch-write 期权,期货,互换 depth:intermediate` — 指定 3 个词条，中级深度
