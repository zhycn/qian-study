对词条内容进行深度质量审查。

如果用户未指定范围，询问：

1. 审查单个词条还是整个目录？
2. 重点关注什么？（准确性、完整性、语言、格式、关联性）

## 审查维度

### 1. 内容准确性

- 金融概念描述是否正确
- 数据/示例是否过时
- 术语使用是否准确
- 法律/监管信息是否准确

### 2. 内容完整性

- 是否覆盖所有必要章节
- 是否有未完成的段落
- 关键信息是否缺失

### 3. 语言一致性

- 是否统一使用中文
- 术语是否前后一致
- 语气是否统一
- 金融术语是否标注英文

### 4. 格式规范性

- 标题层级是否正确
- 链接格式是否统一
- 提示块使用是否恰当

### 5. 通俗性

- 是否使用了生活化类比
- 复杂概念是否解释清楚
- 举例是否贴切易懂

### 6. 关联性

- 是否包含足够的交叉引用
- 延伸阅读是否指向相关专题
- 分类归属是否正确

### 7. SEO 友好性

- frontmatter 的 description 是否准确
- 标题是否包含关键词
- 内容密度是否合理

### 8. 数据溯源

- 文中所有具体数据/数字/统计结果是否标注了来源
- 来源标注格式是否统一（`（来源：机构名称，年份）`）
- 来源机构是否权威可信
- 缺少来源标注的数据逐一列出

## 定量指标（自动计算）

| 指标           | 阈值           | 计算方法                                                   |
| -------------- | -------------- | ---------------------------------------------------------- |
| 总字数         | ≥ 1000 字      | 统计正文（剔除 frontmatter）的中文字数                     |
| 内部链接数     | ≥ 3 个         | 统计 `](/` 或 `](/ 模式的数量                              |
| 英文术语标注   | ≥ 2 处         | 统计 `（英文）` 或 `(English)` 模式的数量                  |
| 章节数         | ≥ 4 个         | 统计 `##` 标题的数量                                       |
| 段落长度       | 平均 ≤ 5 行    | 统计非空段落行数并计算平均值                               |
| 数据来源标注数 | ≥ 文中数据条目 | 统计 `（来源：` 模式的数量，低于文中可识别数据条目数则标记 |

低于阈值的指标标记为提醒，不阻塞通过。

## 输出格式

```markdown
## 审查报告：[词条名]

### 定量指标

- 总字数: X 字（阈值 ≥ 1000）✅/❌
- 内部链接: X 个（阈值 ≥ 3）✅/❌
- 英文术语标注: X 处（阈值 ≥ 2）✅/❌
- 章节数: X 个（阈值 ≥ 4）✅/❌
- 平均段落长度: X 行（阈值 ≤ 5）✅/❌
- 数据来源标注: X 处 ❌/✅

### 严重问题（必须修复）

- [ ] 问题描述

### 中等问题（建议修复）

- [ ] 问题描述

### 轻微问题（可选优化）

- [ ] 问题描述

### 评分

- 准确性: X/5
- 完整性: X/5
- 可读性: X/5
- 通俗性: X/5
- 格式规范: X/5
```

用户确认后可自动修复部分问题。用 Edit 工具修复。

## 工具绑定

| 步骤         | 工具                                |
| ------------ | ----------------------------------- |
| 读取词条文件 | Read                                |
| 统计中文字数 | 用 Bash 的 `wc -m` 过滤 frontmatter |
| 统计链接数   | 用 Grep 查找 `](/` 模式             |
| 统计章节数   | 用 Grep 查找 `##` 标题              |
| 自动修复     | Edit（逐项替换）                    |
| 验证         | `pnpm docs:build`                   |

## 容错

- 文件不存在 → 提示用户确认路径
- 定量指标全部通过 → 在报告开头标注"✅ 全部指标达标"
- 无数据来源可识别 → 将"数据来源标注"标记为"无需标注"
- 词条字数不足 1000 字 → 标记为"内容不足"，但不阻塞

示例：`/ct:review` 或 `/ct:review docs/basics/what-is-finance.md`
