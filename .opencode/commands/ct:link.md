description: 维护词条交叉引用
agent: build
---

维护词条之间的关联和交叉引用。

如果用户未指定任务，询问：

1. 要执行什么操作？（检查断链、添加关联、批量更新）

执行步骤：

1. **检查断链**：
   - 扫描所有词条文件中的 `/glossary/` 链接
   - 验证链接目标文件是否存在
   - 列出不存在的链接

2. **添加关联**：
   - 读取指定词条内容
   - 识别文中提到的相关概念
   - 检查是否已添加对应词条链接
   - 为未链接的概念添加内部链接

3. **批量更新**：
   - 当词条重命名或移动时
   - 更新所有引用该词条的文件
   - 更新 index.md 中的链接
   - 更新 config.mts 中的侧边栏配置

4. **关联推荐**：
   - 分析词条内容主题
   - 推荐应该关联的其他词条
   - 建议在"延伸阅读"中添加的专题目录

输出格式：

```
## 交叉引用报告

### 断链 (3 个)
- `ai.md`: `/glossary/neural-net` → 文件不存在

### 缺失关联 (5 个)
- `llm.md`: 提到"Transformer"但未链接

### 推荐新增延伸阅读
- `mcp.md` → `/agent/framework`
```

用户确认后执行修复。

示例：`/ct:link` 或 `/ct:link --check-broken` 或 `/ct:link docs/glossary/ai.md`
