---
description: 批量深度写作多个词条
agent: build
---

针对多个词条执行批量深度写作全流程：批量识别类型 → 匹配策略 → 大纲 → 写作 → 审查 → 修复。

如果用户未指定范围，询问：

1. 批量处理哪些词条？
   - `empty`：所有仅有 frontmatter 但正文为空的词条
   - `category:xxx`：指定分类的所有词条（如 `category:基础概念`）
   - 用户手动指定列表（以逗号分隔）
2. 目标深度层级是什么？（默认 `beginner`，可选 `intermediate` / `advanced`）
3. 是否覆盖已有内容？（默认不覆盖，仅写入空壳词条）

## 执行步骤

### 步骤 1：确定目标列表

- `empty`：用 Glob 扫描 `docs/**/*.md`，用 Read 读取每个文件，筛选正文字数 < 300 的（使用 `awk '/^---$/{c++; next} c==2' 文件 | wc -m` 统计）
- `category:xxx`：用 Read 读取 `docs/glossary/index.md`，从 `## 分类名` 下方表格提取对应分类的词条链接列
- 手动列表：直接使用用户提供列表

输出目标清单供用户确认，格式：

```
## 目标词条清单（X 个）
1. 词条A（/path）
2. 词条B（/path）
```

### 步骤 2：逐词条执行

对每个目标词条：

1. 用 Read 读取文件，判断类型（从 index.md 分类或 frontmatter）
2. 如果已有正文且未要求覆盖，跳过
3. 生成大纲并写入
4. 撰写正文并写入
5. 执行质量审查（评分维度参考 ct:review：准确性、完整性、可读性、通俗性、格式规范、数据溯源）
6. 自动修复（链接格式、术语标注、数据来源标注）

**每次处理不超过 10 个词条**，超过时分批处理，每批完成后输出状态。单批状态：

```
[1/5] ✅ 词条A（1,200 字，4.2/5）
[2/5] ⏭️ 词条B（已有内容，跳过）
[3/5] ✅ 词条C（2,400 字，4.5/5）
```

### 步骤 3：生成报告

```
## 批量深度写作完成

### 处理概况
- 目标词条：X 个
- 成功写入：X 个
- 跳过（已有内容）：X 个
- 失败：X 个

### 逐词条结果
| 词条 | depth | 字数 | 评分 | 状态 |
|------|-------|------|------|------|
| 词条A | beginner | 1,200 | 4.2/5 | ✅ |
| 词条B | intermediate | 2,400 | 4.5/5 | ✅ |

### 验证
- lint:check ✅/❌
- type-check ✅/❌
- docs:build ✅/❌
```

### 步骤 4：验证

依次运行 `pnpm lint:check`、`pnpm type-check`、`pnpm docs:build`。

### 步骤 5：后续操作

询问用户是否需要：

- 通过 `/vp:commit-push` 提交变更
- 对特定词条执行 `/ct:review` 深度审查
- 对特定词条执行 `/ct:link` 交叉引用补充

## 工具绑定

| 步骤       | 工具                      |
| ---------- | ------------------------- |
| 扫描词条   | Glob + Read               |
| 解析分类   | Read（index.md 表格解析） |
| 逐词条处理 | Read + Write/Edit         |
| 质量审查   | Read + Grep               |
| 自动修复   | Edit                      |
| 验证       | Bash                      |

## 容错

- 单词条写入失败 → 标记失败，继续处理下一个
- 批量中断可恢复，已完成的词条不会丢失
- `docs:build` 失败 → 提示用户查看具体错误，询问是否逐个排查
- 用户可随时中断，已完成的词条文件已写入
- `category:xxx` 解析失败（分类不存在）→ 列出可用分类供用户选择

## 示例

- `/ct:batch-write empty` — 补齐所有空壳词条
- `/ct:batch-write category:新金融概念`
- `/ct:batch-write 期权,期货,互换 depth:intermediate`
