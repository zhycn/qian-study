# 移动支付

> 把钱包装进手机，付钱不用掏现金

## 概述

移动支付（Mobile Payment）是指通过手机等移动设备完成支付交易的技术和服务。主要形式包括扫码支付（QR Code Payment）、NFC 近场支付（Near Field Communication）、App 内支付（In-App Payment）和生物识别支付等。

在中国，支付宝和微信支付已经让"无现金社会"成为现实——从菜市场到奢侈品店，一个二维码搞定一切。2023 年中国移动支付交易规模超过 300 万亿元（来源：中国人民银行），覆盖数亿用户。

## 为什么重要

移动支付深刻**改变了人们的消费习惯和商业形态**：

- **便捷高效**：不用带现金、不用找零，扫码即付，交易时间从分钟级缩短到秒级
- **降低成本**：商家减少现金管理成本和假币风险，POS 机部署成本大幅降低
- **数据沉淀**：每笔交易都有记录，方便记账、分析和精准营销
- **金融普惠**：让没有银行卡的人也能享受数字支付，中国约 8 亿此前无银行账户的人群因此受益（来源：世界银行）
- **生态入口**：移动支付成为超级 App 的流量入口，衍生出理财、信贷、保险等增值服务

## 核心原理

移动支付的技术架构包含三个层次：

**前端交互层**：用户通过手机完成支付操作。

- **二维码支付**：商家展示收款码（主扫）或用户展示付款码（被扫），扫码后发起支付请求
- **NFC 支付**：手机与 POS 机靠近即可完成支付，基于 ISO 14443 标准，支持 Apple Pay、Huawei Pay 等
- **生物识别**：指纹、人脸、声纹等生物特征验证身份，提升支付安全性和便捷性

**支付网关层**：连接用户、商家和银行的中间层。

- **清算机构**：中国银联、网联（NetsUnion）负责跨行交易的清算
- **支付牌照**：第三方支付机构需获得央行支付业务许可证
- **风控系统**：实时监测异常交易，包括设备指纹、行为分析、机器学习模型

**银行结算层**：资金最终通过银行账户体系完成结算。

- **T+1 结算**：大多数移动支付采用次日结算模式
- **备付金管理**：第三方支付机构的客户资金需全额存管在央行指定银行

## 技术/类型详解

**扫码支付**：

- **主扫模式**：用户主动扫描商家二维码，适合小微商户
- **被扫模式**：商家扫描用户付款码，适合超市、便利店等高频场景
- 技术原理：二维码编码支付信息（金额、商户 ID），通过 HTTPS 加密传输到支付服务器

**NFC 支付**：

- 基于 HCE（Host Card Emulation）技术，手机模拟银行卡
- 支持离线支付（小额免密），交易速度更快
- 安全性高于二维码，采用 Tokenization（令牌化）技术保护卡号

**App 内支付**：

- 电商 App、游戏内购等场景
- 通过 SDK 集成支付能力，支持多种支付方式
- Apple 和 Google 对应用内支付收取 15-30% 的平台佣金（来源：Apple/Google 官方政策）

## 实战应用

- **小微商户**：街边摊贩用一张打印的收款码即可收款，零门槛接入
- **公共交通**：全国 300+ 城市支持扫码乘公交地铁（来源：支付宝/微信官方数据）
- **跨境支付**：支付宝和微信支付支持海外商户直接收款，汇率实时结算
- **数字人民币**：与[数字人民币](/posts/digital-yuan)钱包打通，支持双离线支付

## 关键要点

- 主要技术：二维码、NFC、生物识别（指纹、人脸）
- 中国市场由支付宝和微信支付双寡头主导
- 支付安全依赖多层风控体系，包括设备指纹、行为分析、机器学习
- 监管要求：第三方支付机构需持牌经营，客户备付金全额存管

## 常见误区

- **误区一**：移动支付就是扫码支付——移动支付还包括 NFC、App 内支付、生物识别支付等多种形式
- **误区二**：移动支付绝对安全——设备丢失、二维码替换、网络欺诈等风险依然存在
- **误区三**：移动支付会完全取代现金——现金在特定场景（如断网、老年人群）中仍有不可替代的作用
- **误区四**：移动支付数据完全隐私——支付数据会被用于用户画像和精准营销
