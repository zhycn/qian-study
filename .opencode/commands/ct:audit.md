对词条进行质量审查与评分，若评分低于阈值则自动调用 `/ct:deep-write` 重写。

如果用户未指定词条，询问：

1. 审查单个词条还是整个目录？
2. 评分阈值是多少？（默认 3.0/5，低于此值触发 deep-write）
3. 目标深度层级？（触发 deep-write 时使用，默认 beginner）

## 评分标准

综合 6 个维度，每项 1-5 分，取平均分：

| 维度     | 权重 | 评分依据                             |
| -------- | ---- | ------------------------------------ |
| 准确性   | 25%  | 金融概念描述正确，数据无过时         |
| 完整性   | 20%  | 覆盖大纲所有章节，无未完成段落       |
| 可读性   | 20%  | 段落简短（<= 5 行），逻辑清晰        |
| 通俗性   | 15%  | 使用生活化类比，复杂概念解释清楚     |
| 格式规范 | 10%  | 标题层级正确，链接格式统一           |
| 数据溯源 | 10%  | 具体数据标注来源（来源：机构，年份） |

## 定量指标（自动计算）

| 指标         | 阈值            | 计算方法                                         |
| ------------ | --------------- | ------------------------------------------------ |
| 总字数       | >= 1000 字      | `awk '/^---$/{c++; next} c==2' 文件 \| wc -m`    |
| 内部链接数   | >= 3 个         | `grep -c '\]\(/' 文件`                           |
| 英文术语标注 | >= 2 处         | `grep -cP '[（(][A-Za-z][A-Za-z\s-]+[）)]' 文件` |
| 章节数       | >= 4 个         | `grep -c '^## ' 文件`                            |
| 数据来源标注 | >= 文中数据条目 | `grep -c '（来源：' 文件`                        |

低于阈值的指标在报告中列出，影响对应维度评分。

## 执行步骤

### 步骤 1：读取词条

用 Read 读取目标文件。如果文件不存在，提示用户。

### 步骤 2：计算定量指标

按上表逐一计算，记录通过/未通过状态。

### 步骤 3：定性评分

按 6 个维度逐一评分（1-5 分），给出评分理由。

### 步骤 4：计算总分

总分 = 准确性 _ 0.25 + 完整性 _ 0.20 + 可读性 _ 0.20 + 通俗性 _ 0.15 + 格式规范 _ 0.10 + 数据溯源 _ 0.10

### 步骤 5：判断是否需要重写

- 总分 >= 阈值 → 输出审查报告，不触发重写
- 总分 < 阈值 → 输出审查报告，询问用户是否触发 `/ct:deep-write` 重写
  - 用户确认或设置为自动模式 → 调用 `/ct:deep-write` 执行深度写作
  - deep-write 完成后 → 重新执行本命令进行二次审查

## 输出格式

```markdown
## 审查报告：[词条名]

### 定量指标

- 总字数: X 字（阈值 >= 1000）✅/❌
- 内部链接: X 个（阈值 >= 3）✅/❌
- 英文术语标注: X 处（阈值 >= 2）✅/❌
- 章节数: X 个（阈值 >= 4）✅/❌
- 数据来源标注: X 处 ✅/❌

### 定性评分

- 准确性（25%）: X/5 - [理由]
- 完整性（20%）: X/5 - [理由]
- 可读性（20%）: X/5 - [理由]
- 通俗性（15%）: X/5 - [理由]
- 格式规范（10%）: X/5 - [理由]
- 数据溯源（10%）: X/5 - [理由]

### 总分

- 加权平均分: X.XX/5（阈值: X.X）
- 判定: 通过 / 不合格

### 问题清单

- [严重] 问题描述
- [中等] 问题描述
- [轻微] 问题描述

### 后续操作

- 总分 >= 阈值：无需操作
- 总分 < 阈值：已触发 /ct:deep-write 重写（depth: beginner）
```

## 工具绑定

| 步骤            | 工具                       |
| --------------- | -------------------------- |
| 读取词条        | Read                       |
| 定量指标计算    | Bash（awk + grep + wc -m） |
| 定性评分        | Read + 分析                |
| 触发 deep-write | 调用 /ct:deep-write 命令   |
| 二次审查        | 递归调用本命令             |

## 容错

- 文件不存在 → 提示用户确认路径
- 文件仅 frontmatter（正文 < 300 字）→ 直接判定不合格，触发 deep-write
- deep-write 失败 → 报告失败原因，不阻塞其他词条处理
- 二次审查仍不合格 → 标记为"需人工介入"，不无限循环

## 示例

- `/ct:audit docs/basics/what-is-finance.md` — 审查单个词条
- `/ct:audit docs/basics/ --threshold 3.5` — 审查整个目录，阈值 3.5
- `/ct:audit empty --auto-fix` — 审查所有空壳词条，不合格自动重写
