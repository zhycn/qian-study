# 量化交易平台

> 量化交易平台就是"量化投资者的厨房"，给你提供数据、工具和环境，让你能专心做菜（写策略）

## 概述

量化交易平台（Quantitative Trading Platform）是为量化投资者提供一站式服务的在线环境。就像程序员不需要自己造电脑才能写代码一样，量化交易者也不需要自己去买历史数据、搭建回测引擎、连接券商接口——这些平台都帮你准备好了。

一个典型的量化交易平台通常包含以下核心模块：

- **历史数据**：股票、期货、基金等的价格和财务数据，动辄十几年、几千只标的
- **研究环境**：内置 Jupyter Notebook 或在线编辑器，支持 Python 等编程语言
- **回测引擎**：把你的策略放到历史数据中跑，看看过去能赚多少
- **模拟交易**：用实时数据但不花真钱，验证策略在"当下"的表现
- **实盘交易**：部分平台支持直接对接券商，自动下单

## 为什么重要

对于想从零开始做量化的人来说，平台解决了三个核心痛点：

- **数据门槛**：高质量的金融数据非常昂贵，个人购买 Wind/Choice 数据终端年费数万元，平台提供了现成的数据源
- **技术门槛**：回测引擎、交易接口的开发需要大量工程工作，平台帮你封装好了
- **学习曲线**：社区和文档能让你快速上手，不用从零摸索

没有这些平台，个人量化投资者需要自己搞定数据获取、清洗、存储、回测框架、交易接口等一系列工程问题，光搭建环境可能就要几个月。

## 核心原理

### 平台架构

量化交易平台的核心架构通常包含以下层次：

**1. 数据层**（Data Layer）：

- 历史行情数据库（分钟级、日级）
- 财务数据、指数成分、停牌信息等
- 实时行情数据流
- 数据清洗和质量控制

**2. 计算层**（Compute Layer）：

- 回测引擎（事件驱动或向量化）
- 因子计算引擎
- 组合优化引擎
- 风险管理引擎

**3. 接口层**（API Layer）：

- Python SDK / REST API
- 策略开发框架
- 交易接口（券商对接）
- 数据查询接口

**4. 应用层**（Application Layer）：

- 在线研究环境（Notebook）
- 策略管理面板
- 回测结果可视化
- 社区和策略市场

### 回测引擎的两种架构

**事件驱动回测**（Event-Driven Backtesting）：

模拟真实交易流程，按时间顺序处理每一个事件（行情更新、订单成交等）：

```
行情事件 → 策略接收 → 生成订单 → 订单执行 → 成交回报 → 更新持仓
```

- 优点：更接近实盘，能处理复杂的交易逻辑
- 缺点：速度慢，实现复杂
- 代表：Backtrader、QuantConnect

**向量化回测**（Vectorized Backtesting）：

使用数组运算一次性处理整个时间序列：

```python
# 向量化回测示例
signal = (price > ma20).astype(int)  # 1 买入，0 卖出
returns = signal.shift(1) * price.pct_change()
```

- 优点：速度极快，适合大规模因子测试
- 缺点：难以处理复杂的交易逻辑
- 代表：pandas 自建回测、Qlib

### 平台选择决策框架

**1. 按市场选择**：

- A 股 → JoinQuant、RiceQuant、VN.py
- 美股 → QuantConnect、Zipline
- 多市场 → QuantConnect

**2. 按需求选择**：

- 学习入门 → JoinQuant（中文文档友好）
- 策略研究 → Qlib（AI/ML 导向）
- 实盘交易 → VN.py（本地部署、灵活对接）
- 全球市场 → QuantConnect

**3. 按技术能力选择**：

- 编程新手 → BigQuant（可视化搭建）
- 有编程基础 → JoinQuant、RiceQuant
- 资深开发者 → VN.py、自建系统

### 平台的局限性

平台虽然方便，但也有明显的局限性：

**1. 回测与实盘的差距**：

- 平台回测通常使用简化模型，忽略滑点、涨跌停限制、流动性等因素
- 实盘中的订单可能无法按回测价格成交
- 平台数据可能存在幸存者偏差（Survivorship Bias）

**2. 算力限制**：

- 免费平台通常有算力限制（CPU 核心数、内存、运行时间）
- 复杂策略（如大规模因子测试、ML 模型训练）可能跑得很慢
- 付费版本费用不菲

**3. 策略保密性**：

- 在线平台的策略代码存储在第三方服务器上
- 策略泄露风险（虽然平台方通常有保密协议）
- 敏感策略建议本地部署

**4. 数据质量**：

- 平台数据的复权处理可能不准确
- 财务数据可能存在延迟或错误
- 高频数据通常不完整

## 实战应用

### 从零开始的量化学习路径

```
第 1 步：注册 JoinQuant → 跑通第一个策略（双均线）
    ↓
第 2 步：学习 pandas 数据处理 → 自己计算因子
    ↓
第 3 步：尝试多因子策略 → 理解因子合成和加权
    ↓
第 4 步：学习回测陷阱 → 避免过拟合、未来函数
    ↓
第 5 步：模拟交易 → 验证实时表现
    ↓
第 6 步：本地部署 VN.py → 对接实盘
```

### JoinQuant 策略示例

```python
from jqdata import *

def initialize(context):
    # 设定基准
    set_benchmark('000300.XSHG')
    # 设置手续费
    set_order_cost(OrderCost(
        open_tax=0, close_tax=0.001,
        open_commission=0.0003, close_commission=0.0003
    ), type='stock')
    # 每天运行
    run_daily(my_strategy, 'every_bar')

def my_strategy(context):
    # 获取沪深 300 成分股
    stocks = get_index_stocks('000300.XSHG')
    # 简单动量策略：买入过去 20 天涨幅最大的 10 只
    returns = attribute_history(stocks, 20, '1d', 'close', skip_paused=True)
    momentum = returns.iloc[-1] / returns.iloc[0] - 1
    top_stocks = momentum.nlargest(10).index.tolist()
    # 调仓
    order_target_percent(top_stocks, 0.1)
```

### 从平台到本地的迁移

当策略成熟后，可能需要从在线平台迁移到本地：

1. **数据迁移**：从平台导出数据，或使用本地数据源（Tushare、AKShare）
2. **回测迁移**：使用 Backtrader、VN.py 等本地框架重新实现
3. **实盘对接**：通过券商 API 或第三方接口（如 CTP 期货接口）实现自动交易
4. **监控系统**：搭建策略运行监控、异常报警系统

## 关键要点

- 新手建议从 JoinQuant 或 RiceQuant 入手，中文文档友好，A 股数据齐全
- 平台回测结果要谨慎对待，注意是否考虑了滑点、手续费、涨跌停限制
- 免费平台通常有算力限制，复杂策略需要付费版本
- 从研究环境到实盘交易之间还有很大距离，模拟交易是必要的过渡环节
- 不要过度依赖平台内置的"策略库"，别人的策略直接拿来用往往会失效
- 敏感策略建议本地部署，避免策略泄露风险
- 平台是学习工具，不是生产工具——实盘需要更完善的系统

## 常见误区

- **"平台回测 = 实盘收益"**：平台回测往往忽略滑点、涨跌停限制、流动性等实盘因素
- **"免费平台 = 够用"**：免费平台通常有算力限制和数据延迟
- **"直接复制平台策略库"**：别人的策略公开后往往已经失效，盲目复制会亏钱
- **"平台 = 实盘交易"**：从研究环境到实盘交易还有很大距离
- **"所有平台都一样"**：不同平台的数据质量、回测引擎、支持市场差异很大
- **"平台数据没有偏差"**：幸存者偏差、复权处理等问题普遍存在
