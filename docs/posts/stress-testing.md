# 压力测试

压力测试就是给银行做"极限体能测试"——如果 GDP 暴跌 5%、房价腰斩、失业率达到 15%，你的银行还能不能活下去？

## 概述

压力测试（Stress Testing）是**通过模拟极端但可能的经济和市场情景，评估金融机构或金融体系在压力条件下的稳健性**的风险管理工具。

压力测试的三大类型：

- **敏感性分析**（Sensitivity Analysis）：测试单一变量变化的影响，如利率上升 200 个基点
- **情景分析**（Scenario Analysis）：测试多个变量同时变化的综合影响，如经济衰退 + 股市暴跌 + 房价下跌
- **反向压力测试**（Reverse Stress Testing）：从"机构倒闭"的结果反推，找出导致倒闭的情景

压力测试的关键要素：

| 要素 | 说明 |
|------|------|
| 情景设计 | 定义压力情景，包括宏观经济变量和市场变量 |
| 传导模型 | 将情景变化转化为财务影响（损失、收入变化） |
| 资本评估 | 评估压力下的资本充足率是否满足最低要求 |
| 应对措施 | 如果资本不足，制定补充资本或缩减业务的计划 |

历史上的重要压力测试：

- 2009 年美国 SCAP（Supervisory Capital Assessment Program）— 金融危机后首次系统性银行压力测试
- 2011 年欧盟 EBA 压力测试 — 欧债危机期间
- 2014 年美联储 CCAR（Comprehensive Capital Analysis and Review）— 年度常态化压力测试
- 2020 年疫情特别压力测试 — 评估新冠疫情对银行体系的影响

## 为什么重要

- **危机教训**：2008 年金融危机前，很多银行在"正常"模型下看起来很健康，但在极端情景下迅速崩溃。压力测试弥补了这一盲点
- **监管要求**：巴塞尔协议 III 和各国监管机构将压力测试纳入法定要求
- **市场信心**：公开压力测试结果可以增强市场对银行体系的信心
- **内部管理**：压力测试帮助银行识别脆弱点，提前制定应对计划
- **资本规划**：压力测试结果直接影响银行的分红和股票回购计划

## 核心原理

### 压力测试的基本框架

压力测试的核心逻辑：

$$\text{资本}_{\text{期末}} = \text{资本}_{\text{期初}} + \text{净收入} - \text{损失} - \text{分红}$$

在压力情景下：
- 净收入下降（贷款需求减少、交易收入下降）
- 损失上升（贷款违约增加、投资减值）
- 如果资本期末低于监管最低要求，银行需要补充资本

### 信用风险压力测试

贷款损失是压力测试中最重要的组成部分。

预期损失（Expected Loss, EL）公式：

$$EL = PD \times LGD \times EAD$$

其中：
- **PD**（Probability of Default）：违约概率
- **LGD**（Loss Given Default）：违约损失率
- **EAD**（Exposure at Default）：违约风险敞口

在压力情景下：
- PD 上升：经济衰退导致更多借款人违约
- LGD 上升：抵押品价值下降，回收率降低
- EAD 可能变化：信贷额度使用率上升

### 市场风险压力测试

交易账户的市场风险压力测试通常使用：

**历史情景法**：重现历史极端事件的影响
- 2008 年雷曼倒闭
- 2020 年 3 月疫情暴跌
- 1998 年 LTCM 危机

**假设情景法**：构造可能但尚未发生的极端事件
- 中美脱钩
- 主要主权债务违约
- 房地产市场崩盘

## 模型/方法详解

### 美联储 CCAR 框架

CCAR（Comprehensive Capital Analysis and Review）是美国最严格的银行压力测试：

**参与机构**：资产超过 1000 亿美元的银行控股公司

**情景类型**（每年更新）：
- **基线情景**（Baseline）：经济共识预测
- **不利情景**（Adverse）：温和衰退
- **严重不利情景**（Severely Adverse）：严重衰退 + 市场冲击

**严重不利情景示例**（2023 年）：
- GDP 下降 4.5%
- 失业率升至 10%
- 房价下降 25%
- 商业地产价格下降 35%
- 股市下跌 50%

**评估指标**：
- 最低资本比率（CET1 ≥ 4.5%）
- 压力资本缓冲（SCB）
- 补充杠杆率（SLR ≥ 3%）

### 欧洲银行管理局（EBA）框架

EBA 压力测试的特点：

- **自下而上**（Bottom-up）：各银行用自己的模型计算，EBA 提供统一情景
- **静态资产负债表**：假设银行在测试期间不调整资产负债结构
- **透明度要求**：公布各银行详细结果

### 反向压力测试

反向压力测试的逻辑与传统压力测试相反：

1. 定义"失败"状态（如 CET1 比率降至 3% 以下）
2. 反向推导：什么样的情景会导致这种结果？
3. 评估这些情景的发生概率
4. 制定预防措施

反向压力测试特别适合识别**非线性风险**和**尾部风险**。

## 主流方案对比

| 方法 | 优势 | 局限 | 适用场景 |
|------|------|------|----------|
| 敏感性分析 | 简单直观，易于理解 | 忽略变量间相关性 | 初步风险评估 |
| 历史情景法 | 基于真实事件，可信度高 | 历史不一定重演 | 市场风险 |
| 假设情景法 | 可以测试前所未见的风险 | 主观性强 | 新兴风险 |
| 反向压力测试 | 识别极端尾部风险 | 结果可能过于极端 | 系统性风险 |
| 宏观压力测试 | 评估整个金融体系 | 模型复杂，数据要求高 | 金融稳定 |

## 实战应用

### 压力测试的实施步骤

1. **确定范围**：哪些业务线、哪些风险类型纳入测试
2. **设计情景**：基于宏观经济预测和历史极端事件
3. **数据准备**：收集贷款组合、交易头寸、抵押品等数据
4. **模型运行**：将情景输入风险模型，计算损失和资本变化
5. **结果分析**：识别脆弱点，评估是否需要采取行动
6. **报告与沟通**：向管理层、董事会和监管机构报告
7. **行动计划**：制定资本补充、风险缓释措施

### 压力测试的挑战

**模型风险**：
- 压力测试依赖模型，模型本身可能有缺陷
- 2008 年危机前，很多模型低估了房价下跌的幅度

**数据质量**：
- 压力测试需要大量历史数据
- 极端事件数据稀少，模型外推不可靠

**行为反应**：
- 压力测试假设资产负债表不变，但银行在压力下会调整行为
- 集体行为可能加剧压力（如同时抛售资产）

**情景设计的主观性**：
- 什么样的情景算"极端但可能"？
- 不同监管机构的标准不同

### 气候压力测试（新兴领域）

近年来，气候风险压力测试成为新热点：

- **物理风险**：极端天气事件对抵押品和业务的影响
- **转型风险**：碳税、监管变化对高碳资产的影响
- **时间跨度**：气候风险的影响可能在 10-30 年后显现，远超传统压力测试的时间范围

英格兰银行（Bank of England）和欧洲央行（ECB）已率先开展气候压力测试。

## 关键要点

- 压力测试不是预测未来，而是评估"如果发生 X，会怎样"
- 压力测试的结果取决于情景设计和模型假设，不是绝对真理
- 压力测试应该与日常管理结合，而不是应付监管的"年度作业"
- 反向压力测试是识别未知风险的重要补充工具
- 气候风险压力测试代表了压力测试的新前沿

## 常见误区

- **误区一**："压力测试通过了就安全了"——压力测试只测试了预设情景，未测试的情景可能更危险
- **误区二**："压力测试就是预测危机"——压力测试是评估韧性，不是预测。它回答"能不能扛住"，不是"会不会发生"
- **误区三**："越极端的情景越好"——过于极端的情景可能失去参考价值，关键是"极端但可能"
- **误区四**："压力测试只是监管要求"——压力测试是风险管理的核心工具，不仅是合规要求
- **误区五**："模型越复杂越准确"——复杂模型可能产生虚假的精确感，简单透明的模型有时更有用
