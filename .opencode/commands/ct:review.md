---
description: 词条质量审查
agent: build
---

对词条内容进行深度质量审查。

如果用户未指定范围，询问：

1. 审查单个词条还是整个目录？
2. 重点关注什么？（准确性、完整性、语言、格式、关联性）

审查维度：

1. **内容准确性**
   - 技术描述是否正确
   - 数据/示例是否过时
   - 术语使用是否准确

2. **内容完整性**
   - 是否覆盖所有必要章节
   - 是否有未完成的段落
   - 关键信息是否缺失

3. **语言一致性**
   - 是否统一使用中文
   - 术语是否前后一致
   - 语气是否统一
   - 技术术语是否标注英文

4. **格式规范性**
   - 标题层级是否正确
   - 代码块是否标注语言
   - 链接格式是否统一
   - 提示块使用是否恰当

5. **关联性**
   - 是否包含足够的交叉引用
   - 延伸阅读是否指向相关专题
   - 分类归属是否正确

6. **SEO 友好性**
   - frontmatter 的 description 是否准确
   - 标题是否包含关键词
   - 内容密度是否合理

输出格式：

```
## 审查报告：[词条名]

### 严重问题 (必须修复)
- [ ] 问题描述

### 中等问题 (建议修复)
- [ ] 问题描述

### 轻微问题 (可选优化)
- [ ] 问题描述

### 评分
- 准确性: X/5
- 完整性: X/5
- 可读性: X/5
- 格式规范: X/5
```

用户确认后可自动修复部分问题。

示例：`/ct:review` 或 `/ct:review docs/glossary/mcp.md`
