# 统计套利

统计套利就是发现"同样的东西卖不同的价"——两瓶一样的矿泉水，一瓶 2 块一瓶 5 块，你买便宜的等价格回归正常时再卖出，赚中间的差价。

## 概述

统计套利（Statistical Arbitrage，简称 StatArb）是**利用统计方法识别资产价格的异常偏离，并押注价格回归正常关系的量化策略**。

最经典的统计套利策略是**配对交易（Pairs Trading）**：

1. 找到两只历史价格走势高度相关的股票（比如可口可乐和百事可乐）
2. 当两只股票的价格差偏离历史均值时，做空涨得多的，做多涨得少的
3. 等价格差回归正常时平仓，赚取差价

统计套利的核心假设：**历史统计关系在未来会延续**。但这个假设并非永远成立——当公司基本面发生结构性变化时，历史关系可能永久打破。

## 为什么重要

- **市场中性**：统计套利通常同时做多和做空，对冲掉市场方向性风险
- **低相关性**：与市场涨跌相关性低，是组合分散化的好工具
- **高频化趋势**：随着算法和算力提升，统计套利从日频发展到毫秒级
- **机构标配**：对冲基金和量化基金的核心策略之一
- **策略容量大**：相比高频做市，统计套利的策略容量更大，适合中等规模资金

## 核心原理

统计套利的数学基础是**协整理论（Cointegration Theory）**。

两个时间序列 $X_t$ 和 $Y_t$ 各自可能是非平稳的（随机游走），但它们的线性组合可能是平稳的：

$$Y_t = \alpha + \beta X_t + \varepsilon_t$$

如果残差 $\varepsilon_t$ 是平稳的（均值回归），则 $X$ 和 $Y$ 存在协整关系。

### 协整检验

常用的协整检验方法是 **Engle-Granger 两步法**：

**第一步**：用 OLS 回归估计协整向量 $\beta$：

$$Y_t = \alpha + \beta X_t + \varepsilon_t$$

**第二步**：对残差 $\varepsilon_t$ 进行 ADF（Augmented Dickey-Fuller）单位根检验：

$$\Delta \varepsilon_t = \gamma \varepsilon_{t-1} + \sum_{i=1}^{p} \delta_i \Delta \varepsilon_{t-i} + u_t$$

如果 $\gamma < 0$ 且在统计上显著，则残差是平稳的，$X$ 和 $Y$ 存在协整关系。

### 价差建模

价差（Spread）$S_t = Y_t - \beta X_t$ 通常被建模为 Ornstein-Uhlenbeck（OU）过程：

$$dS_t = \theta(\mu - S_t)dt + \sigma dW_t$$

其中：

- $\theta$：均值回归速度（越大回归越快）
- $\mu$：长期均值
- $\sigma$：波动率
- $W_t$：布朗运动

OU 过程的半均值回复时间（Half-life）为：

$$\text{Half-life} = \frac{\ln(2)}{\theta}$$

半均值回复时间告诉你，价差偏离均值后，需要多长时间才能回归一半。这是决定持仓周期的关键参数。

## 技术/方法详解

### 配对选择方法

**1. 距离法（Distance Method）**

计算所有股票对之间的历史价格距离，选择距离最小的配对。距离定义为：

$$D_{ij} = \sum_{t=1}^{T} \left| \frac{P_{i,t}}{P_{i,0}} - \frac{P_{j,t}}{P_{j,0}} \right|^2$$

**2. 协整法（Cointegration Method）**

对候选股票对进行协整检验，选择 ADF 统计量最显著（p 值最小）的配对。

**3. 主成分法（PCA Method）**

对股票收益率矩阵做主成分分析，提取共同因子。残差（特异收益）均值回归的股票对是潜在的套利标的。

**4. 机器学习法**

使用聚类算法（如 DBSCAN、层次聚类）自动发现相关性高的股票组。

### 交易信号生成

基于价差的 Z-score 生成交易信号：

$$Z_t = \frac{S_t - \mu}{\sigma}$$

交易规则：

- $Z_t > Z_{\text{entry}}$（如 2.0）：做空价差（做空 $Y$，做多 $X$）
- $Z_t < Z_{\text{exit}}$（如 0.5）：平仓
- $Z_t < -Z_{\text{entry}}$（如 -2.0）：做多价差（做多 $Y$，做空 $X$）
- $Z_t > -Z_{\text{exit}}$（如 -0.5）：平仓

### 仓位管理

统计套利的仓位通常基于价差的波动率来设定：

$$\text{Position Size} = \frac{\text{Target Volatility}}{\sigma_{\text{spread}}}$$

这样可以确保不同配对的风险贡献一致。

## 主流方案对比

| 方法            | 适用场景     | 优点               | 缺点                     |
| --------------- | ------------ | ------------------ | ------------------------ |
| **距离法**      | 快速筛选     | 简单直观，计算快   | 不保证协整关系           |
| **协整法**      | 精确配对     | 统计严谨           | 计算量大，需要长历史数据 |
| **主成分法**    | 多资产组合   | 可以处理大量股票   | 经济解释性弱             |
| **Kalman 滤波** | 动态对冲比率 | 自适应调整 $\beta$ | 实现复杂                 |
| **机器学习**    | 非线性关系   | 捕捉复杂模式       | 容易过拟合，可解释性差   |

### 从配对交易到多腿统计套利

现代统计套利已经从简单的"两只股票配对"发展为"多腿组合"：

- **行业中性统计套利**：在同一行业内构建多空组合
- **因子中性统计套利**：在控制因子暴露后，利用残差收益的均值回归
- **跨市场统计套利**：利用相关市场（如 A 股和 H 股、期货和现货）的价差
- **ETF 套利**：利用 ETF 价格与净值（NAV）的偏离

## 实战应用

### Python 实现示例

```python
import numpy as np
import pandas as pd
from scipy import stats

# 1. 计算价差
spread = stock_y - hedge_ratio * stock_x

# 2. 计算 Z-score
z_score = (spread - spread.mean()) / spread.std()

# 3. 生成信号
signal = np.zeros(len(z_score))
signal[z_score > 2.0] = -1  # 做空价差
signal[z_score < -2.0] = 1   # 做多价差
signal[np.abs(z_score) < 0.5] = 0  # 平仓

# 4. 计算收益
returns = signal.shift(1) * spread.pct_change()
```

### 关键参数选择

| 参数     | 典型值                          | 说明                               |
| -------- | ------------------------------- | ---------------------------------- |
| 回看窗口 | 60-252 天 | 用于估计均值和标准差               |
| 入场阈值 | 1.5-2.5σ  | 太低交易太频繁，太高机会太少       |
| 出场阈值 | 0.3-0.8σ  | 太低可能错过利润，太高持仓时间过长 |
| 止损阈值 | 3.5-5.0σ  | 防止价差永久偏离                   |
| 调仓频率 | 日频/分钟频                     | 取决于数据频率和交易成本           |

### 风险控制

统计套利不是无风险的，主要风险包括：

- **模型风险**：协整关系可能在未来失效
- **流动性风险**：做空标的可能流动性不足
- **执行风险**：两只股票的成交可能不同步
- **尾部风险**：极端事件下价差可能大幅偏离
- **融券风险**：做空需要融券，可能面临"逼空"（Short Squeeze）

## 关键要点

- 统计套利不等于"无风险套利"，历史关系可能在未来失效
- 协整关系是统计套利的数学基础，ADF 检验是核心工具
- OU 过程的半均值回复时间是决定持仓周期的关键参数
- 2007 年 8 月的"量化危机"就是统计套利策略集体平仓导致的踩踏
- 交易成本（手续费、滑点、融券成本）是统计套利的"隐形杀手"
- 止损机制必不可少——价差可能永远不回归
- A 股融券限制和 T+1 制度制约了统计套利的实施

## 常见误区

- **"统计套利 = 无风险套利"**：历史关系可能在未来失效
- **"个人投资者可以做"**：需要大量数据和算力支持，个人投资者难以与机构竞争
- **"资金规模不影响"**：策略容量有限，资金规模太大会影响价格关系本身
- **"交易成本可以忽略"**：手续费、滑点、融券成本是统计套利的"隐形杀手"
- **"协整关系 = 永久有效"**：协整关系可能因公司基本面变化、行业结构调整而失效
- **"配对越多越好"**：配对数量增加会降低单个配对的资金分配，但也会增加管理复杂度
- **"A 股统计套利与美股一样"**：A 股融券限制和 T+1 制度制约了统计套利的实施，ETF 套利和可转债套利是更可行的方式

## 中国视角

A 股统计套利的特殊性：

- **融券限制**：A 股融券券源有限，热门标的经常"一券难求"，制约了配对交易的实施。2024 年监管进一步收紧融券业务
- **T+1 限制**：当天买入次日才能卖出，无法实现日内统计套利
- **ETF 套利**：A 股 ETF 与成分股之间的套利是更可行的统计套利方式，如沪深 300ETF 与期货之间的基差套利
- **AH 股溢价**：同一家公司在 A 股和 H 股的价格差异是天然的统计套利标的，但受资本管制限制，套利空间有限
- **可转债套利**：A 股可转债与正股之间的统计关系是特色套利机会，但需关注强赎条款和转股溢价率

对普通投资者：统计套利对数据、算力、执行速度要求高，个人投资者难以与机构竞争。建议通过量化基金间接参与。
